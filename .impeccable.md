## Design Context

### Users
Lirah serves both power developers who live in the terminal and developers exploring AI CLI agents. The primary context is focused development work — composing prompts, managing file context, tracking token budgets, and navigating codebases. The interface should feel like a precision instrument that rewards expertise while remaining approachable to newcomers. Users are keyboard-driven and value speed over polish.

### Brand Personality
**Technical, precise, sharp.**

Lirah is an engineering tool, not a consumer product. The OCR monospace font, dashed "sketch" borders, and dark-first palette communicate that this is built by developers, for developers. Every element should feel intentional and measured — no decoration for decoration's sake.

### Aesthetic Direction
- **Visual tone:** Dense, information-rich, terminal-native. Inspired by Linear/Raycast (sleek, keyboard-driven productivity) and VS Code/Zed (panel-based, tree-navigated, dense information display).
- **Anti-references:** Electron-heavy apps with sluggish UI, consumer SaaS dashboards with excessive whitespace, playful/rounded aesthetics (Notion, Slack). Nothing should feel slow or bloated.
- **Theme:** Dark-first with 17+ themes. Default is Tokyo Night. The theme system is comprehensive — terminal colors, UI semantic colors, flowchart colors all coordinated per theme.
- **Typography:** Typestar OCR as primary monospace font, compact scale (10-16px). Everything is monospace — this is a terminal app.
- **Borders:** Dashed "engineering sketch" style borders at low opacity. Sharp corners (small radii: 0.5rem max).
- **Spacing:** Compact and dense. Maximize information density without feeling cramped.
- **Scrollbars:** Thin (6px), translucent, minimal.
- **Transitions:** Subtle 150ms eases for color/background changes. No flashy animations.

### Design Principles

1. **Terminal-first** — The terminal is the primary interface. Everything else (sidebar, textarea, dialogs) enhances it, never competes with it. Design decisions should preserve terminal focus.

2. **Information density over whitespace** — Show more, scroll less. Use compact spacing, small fonts, and dense layouts. Every pixel should earn its place. Developers prefer scanning dense data to clicking through pages.

3. **Keyboard-driven** — Every action should be reachable by keyboard. Visual affordances for mouse users are secondary to keyboard shortcuts. Shortcuts should be discoverable but not visually dominant.

4. **Theme-coherent** — All UI elements must respect the active theme's semantic color system. Never hardcode colors. Use CSS custom properties and Tailwind theme tokens so every theme looks intentional, not bolted on.

5. **Precision over personality** — Favor sharp, measured, technical aesthetics. The OCR font and dashed borders are the personality — layer on restraint, not flair. When in doubt, remove rather than add.
